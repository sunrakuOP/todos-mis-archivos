## 📂 Estructura de carpetas

```
proyectos/
├── README.md
├── ideas/          ← Ideas de proyectos futuros (notas)
└── terminados/     ← Proyectos completados
```

### Estructura recomendada para cada proyecto

```
proyectos/terminados/<nombre-proyecto>/
├── README.md       ← Descripción, tecnologías, cómo ejecutarlo
├── src/            ← Código fuente
├── assets/         ← Imágenes, estilos, recursos
└── docs/           ← Documentación extra (opcional)
```

---

## 💡 Cómo documentar un proyecto

Cada proyecto debe tener un `README.md` que incluya:

1. **Nombre y descripción** — ¿Qué hace el proyecto?
2. **Tecnologías usadas** — Lenguajes, frameworks, librerías
3. **Cómo ejecutarlo** — Pasos para correr el proyecto localmente
4. **Capturas de pantalla** (si aplica)
5. **Lo que aprendí** — Reflexión personal sobre el proyecto
